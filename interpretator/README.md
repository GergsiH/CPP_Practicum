# Проект: Интерпретатор языка программирования на C++

В этом проекте я разработала интерпретатор для простого императивного языка программирования с поддержкой переменных, условных операторов, циклов и функций.

## Что я реализовала

### 1. Лексический и синтаксический анализ
Создала сканер для разбиения исходного кода на токены (идентификаторы, ключевые слова, операторы, литералы) и парсер для построения абстрактного синтаксического дерева. Реализовала грамматику с поддержкой арифметических и логических выражений, объявления переменных и составных операторов.

### 2. Система типов и управление памятью
Разработала систему динамической типизации с поддержкой основных типов данных (целые, вещественные, строки, булевы значения). Реализовала таблицу символов для хранения переменных и их значений с областями видимости, включая механизм замыканий для функций.

### 3. Выполнение программ и обработка ошибок
Написала интерпретатор, который обходит синтаксическое дерево и выполняет инструкции. Добавила поддержку условных операторов (if/else), циклов (while, for), функций с параметрами и возвращаемыми значениями. Реализовала систему обработки ошибок с информативными сообщениями о синтаксических, семантических и runtime-ошибках.

### 4. Стандартная библиотека и отладка
Создала набор встроенных функций для ввода/вывода, математических операций и работы со строками. Добавила режим отладки с пошаговым выполнением, выводом значений переменных и трассировкой стека вызовов. Реализовала возможность загрузки кода из файлов и интерактивного режима (REPL).

## Особенности реализации и технологии
Проект написан на C++ с использованием паттерна Visitor для обхода AST и хранения состояния в стеке вызовов. Лексический анализ реализован через конечный автомат, синтаксический — рекурсивным спуском с восстановлением после ошибок. Для хранения переменных используется комбинация хэш-таблиц и стека для поддержки вложенных областей видимости. Интерпретатор включает сборку мусора через подсчёт ссылок и оптимизацию хвостовой рекурсии. Система поддерживает многопоточность через изоляцию контекстов выполнения и имеет модульную архитектуру для легкого расширения новыми конструкциями языка.
